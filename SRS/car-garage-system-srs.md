# เอกสารข้อกำหนดความต้องการซอฟต์แวร์ (SRS)
## ระบบบริหารจัดการอู่ซ่อมรถยนต์ขนาดเล็ก
### เวอร์ชัน 1.0

## 1. บทนำ
### 1.1 วัตถุประสงค์
เอกสารนี้อธิบายข้อกำหนดความต้องการซอฟต์แวร์สำหรับระบบบริหารจัดการอู่ซ่อมรถยนต์ขนาดเล็ก โดยมีวัตถุประสงค์เพื่อ:
- จัดการข้อมูลรถและลูกค้า
- อำนวยความสะดวกในการนัดหมายและรับรถเข้าซ่อม
- ติดตามงานซ่อมและสถานะ
- จัดการอะไหล่และสต็อก
- คำนวณค่าบริการและค่าอะไหล่
- สร้างรายงานและวิเคราะห์ข้อมูล
- เพิ่มประสิทธิภาพการบริหารจัดการอู่

### 1.2 ขอบเขต
ระบบจะครอบคลุมการทำงานหลักดังนี้:
- การจัดการข้อมูลลูกค้าและรถ
- การนัดหมายและจองคิว
- การรับรถเข้าซ่อม
- การวินิจฉัยปัญหา
- การจัดการงานซ่อม
- การจัดการอะไหล่และสต็อก
- การคำนวณค่าบริการ
- การส่งมอบรถ
- การออกรายงาน

### 1.3 ลักษณะอู่ที่รองรับ
ระบบนี้เหมาะสำหรับอู่ขนาดเล็ก ที่มีลักษณะ:
- พนักงาน 5-15 คน
- ช่างซ่อม 3-8 คน
- คิวซ่อมไม่เกิน 30 คันต่อวัน
- ไม่ซับซ้อนมาก จัดการง่าย

## 2. ความต้องการด้านฟังก์ชันการทำงาน
### 2.1 การจัดการข้อมูลลูกค้าและรถ
#### 2.1.1 การจัดการข้อมูลลูกค้า
1. ข้อมูลลูกค้า:
   - รหัสลูกค้า (สร้างอัตโนมัติ)
   - ชื่อ-นามสกุล
   - เลขบัตรประชาชน
   - เบอร์โทรศัพท์
   - อีเมล
   - Line ID
   - ที่อยู่
   - วันเกิด
   - ประเภทลูกค้า (ทั่วไป, VIP, บริษัท)
   - วันที่ลงทะเบียน

2. ข้อมูลผู้ติดต่อฉุกเฉิน:
   - ชื่อผู้ติดต่อ
   - ความสัมพันธ์
   - เบอร์โทรศัพท์

3. ประวัติการใช้บริการ:
   - รถที่เคยซ่อม
   - วันที่ซ่อมล่าสุด
   - ยอดเงินค้างชำระ
   - จำนวนครั้งที่ใช้บริการ
   - มูลค่าการใช้บริการรวม

#### 2.1.2 การจัดการข้อมูลรถ
1. ข้อมูลรถยนต์:
   - รหัสรถ (สร้างอัตโนมัติ)
   - เจ้าของรถ (ลิงก์กับข้อมูลลูกค้า)
   - ทะเบียนรถ
   - ยี่ห้อ (Toyota, Honda, Mazda, ฯลฯ)
   - รุ่น
   - ปีที่ผลิต
   - สี
   - เลขตัวถังรถ (VIN)
   - เลขเครื่องยนต์
   - เลขไมล์ปัจจุบัน
   - ประเภทเชื้อเพลิง (เบนซิน, ดีเซล, ไฮบริด)
   - ประเภทเกียร์ (ธรรมดา, อัตโนมัติ)
   - รูปภาพรถ

2. ข้อมูลประกันภัย:
   - ประเภทประกัน (ชั้น 1, 2, 3, พ.ร.บ.)
   - บริษัทประกัน
   - เลขกรมธรรม์
   - วันหมดอายุ

3. ประวัติการซ่อม:
   - วันที่เข้าซ่อม
   - อาการ/ปัญหา
   - งานที่ทำ
   - อะไหล่ที่เปลี่ยน
   - เลขไมล์ขณะซ่อม
   - ค่าใช้จ่าย
   - ช่างผู้รับผิดชอบ

4. การเตือนการบำรุงรักษา:
   - เปลี่ยนถ่ายน้ำมันเครื่อง (ทุก 5,000 km)
   - เปลี่ยนผ้าเบรก (ตามสภาพ)
   - ตรวจเช็คต่อภาษี
   - ตรวจเช็คประกันภัย

### 2.2 การนัดหมายและจองคิว
#### 2.2.1 การนัดหมาย
1. ช่องทางการนัดหมาย:
   - โทรศัพท์
   - เว็บไซต์/แอปพลิเคชัน
   - Line Official
   - หน้าร้าน
   - Walk-in (ไม่นัด)

2. ข้อมูลการนัดหมาย:
   - รหัสนัดหมาย
   - ข้อมูลลูกค้า
   - ข้อมูลรถ
   - วันที่นัดหมาย
   - เวลานัดหมาย
   - ประเภทงานซ่อม (เปลี่ยนถ่ายน้ำมัน, เปลี่ยนยาง, ซ่อมเครื่องยนต์, ฯลฯ)
   - รายละเอียดปัญหาเบื้องต้น
   - ประมาณเวลาที่ใช้
   - ช่างที่รับผิดชอบ
   - สถานะนัดหมาย (ยืนยัน, รอยืนยัน, ยกเลิก)

3. การจัดการนัดหมาย:
   - เพิ่มนัดหมายใหม่
   - แก้ไขนัดหมาย
   - ยกเลิกนัดหมาย
   - ยืนยันนัดหมาย
   - เลื่อนนัดหมาย

4. การแจ้งเตือน:
   - แจ้งเตือนลูกค้าก่อนนัด 1 วัน
   - แจ้งเตือนลูกค้าก่อนนัด 1 ชั่วโมง
   - แจ้งเตือนช่างก่อนเวลานัด
   - SMS/Line/Email

#### 2.2.2 ปฏิทินและตารางคิว
1. ปฏิทินการนัดหมาย:
   - แสดงรายการนัดหมายทั้งหมด
   - มุมมองรายวัน/รายสัปดาห์/รายเดือน
   - แสดงสีตามสถานะ
   - แสดงช่างที่รับผิดชอบ
   - Drag & Drop เปลี่ยนเวลา

2. การจัดการคิว:
   - คิวรอเข้าซ่อม
   - คิวกำลังซ่อม
   - คิวรอรับรถ
   - เวลาโดยประมาณ
   - หน้าจอแสดงคิว (TV Display)

### 2.3 การรับรถเข้าซ่อม
#### 2.3.1 ขั้นตอนการรับรถ
1. Check-in รถเข้าอู่:
   - ค้นหาข้อมูลลูกค้า/รถ
   - สแกน QR Code นัดหมาย
   - บันทึกเลขไมล์
   - บันทึกระดับน้ำมันเชื้อเพลิง
   - บันทึกอาการเบื้องต้น
   - ตรวจสอบสภาพภายนอก
   - ถ่ายรูปรถรอบคัน (4-8 มุม)
   - ถ่ายรูปรอยขีดข่วน/ชำรุด
   - บันทึกของในรถ

2. ใบรับรถ (Job Order):
   - รหัสงาน (สร้างอัตโนมัติ)
   - วันที่-เวลารับรถ
   - ข้อมูลรถและลูกค้า
   - เลขไมล์
   - ระดับเชื้อเพลิง
   - อาการที่แจ้ง
   - รูปภาพรถ
   - ลายเซ็นลูกค้า (ดิจิทัล)
   - พนักงานรับรถ

3. การมอบหมายงาน:
   - เลือกช่างรับผิดชอบ
   - ประเภทงาน
   - ประมาณเวลาที่ใช้
   - ประมาณค่าใช้จ่าย
   - วันเวลาโดยประมาณที่เสร็จ

#### 2.3.2 การวินิจฉัยปัญหา
1. การตรวจสอบเบื้องต้น:
   - ช่างตรวจสอบสภาพรถ
   - วินิจฉัยปัญหา
   - บันทึกรายละเอียดปัญหา
   - ถ่ายรูป/วิดีโอประกอบ
   - ระบุสาเหตุ

2. การประเมินงานซ่อม:
   - รายการงานที่ต้องทำ
   - อะไหล่ที่ต้องเปลี่ยน
   - ค่าแรง
   - ค่าอะไหล่
   - รวมค่าใช้จ่าย
   - ระยะเวลาที่ใช้

3. การขออนุมัติ:
   - สร้างใบเสนอราคา
   - ส่งให้ลูกค้าอนุมัติ
   - รอการตอบรับ
   - บันทึกการอนุมัติ

### 2.4 การจัดการงานซ่อม
#### 2.4.1 ประเภทงานซ่อม
1. งานบำรุงรักษาทั่วไป:
   - เปลี่ยนถ่ายน้ำมันเครื่อง
   - ตรวจเช็คระบบเบรก
   - ตรวจสอบระบบไฟฟ้า
   - ล้างแอร์
   - ตั้งศูนย์ถ่วงล้อ
   - เปลี่ยนยาง

2. งานซ่อมแซม:
   - ซ่อมเครื่องยนต์
   - ซ่อมเกียร์
   - ซ่อมช่วงล่าง
   - ซ่อมระบบไฟฟ้า
   - ซ่อมแอร์
   - เคาะพ่นสี

3. งานติดตั้งอุปกรณ์:
   - ติดตั้งกล้องหลัง
   - ติดตั้งเครื่องเสียง
   - ติดฟิล์มกรองแสง
   - ติดตั้ง GPS

#### 2.4.2 การบันทึกงานซ่อม
1. รายละเอียดงาน:
   - รหัสงานย่อย
   - ประเภทงาน
   - รายละเอียดงาน
   - ช่างผู้ทำ
   - เวลาเริ่ม
   - เวลาเสร็จ
   - สถานะงาน (รอทำ, กำลังทำ, เสร็จแล้ว, รอตรวจสอบ)

2. อะไหล่ที่ใช้:
   - รายการอะไหล่
   - จำนวน
   - ราคาต่อหน่วย
   - ราคารวม
   - เบิกจากสต็อก/สั่งซื้อ

3. ค่าแรง:
   - ค่าแรงช่าง
   - ชั่วโมงการทำงาน
   - อัตราค่าแรงต่อชั่วโมง
   - รวมค่าแรง

4. บันทึกการทำงาน:
   - รายละเอียดสิ่งที่ทำ
   - ปัญหาที่พบเพิ่มเติม
   - รูปภาพก่อน-หลัง
   - ข้อแนะนำ

#### 2.4.3 การติดตามสถานะ
1. สถานะงานซ่อม:
   - รอช่างตรวจสอบ
   - รอลูกค้าอนุมัติ
   - รอสั่งอะไหล์
   - กำลังซ่อม
   - รอตรวจสอบ
   - เสร็จสิ้น
   - รอรับรถ

2. Timeline การซ่อม:
   - แสดงขั้นตอนการทำงาน
   - เวลาแต่ละขั้นตอน
   - ผู้รับผิดชอบ
   - อัพเดตแบบ Real-time

3. การแจ้งเตือนลูกค้า:
   - แจ้งเมื่อเริ่มซ่อม
   - แจ้งความคืบหน้า
   - แจ้งเมื่อเสร็จ พร้อมรับรถ
   - แจ้งค่าใช้จ่ายเพิ่มเติม (ถ้ามี)

### 2.5 การจัดการอะไหล่และสต็อก
#### 2.5.1 การจัดการอะไหล่
1. ข้อมูลอะไหล่:
   - รหัสอะไหล่
   - ชื่ออะไหล่
   - ยี่ห้อรถที่ใช้ได้
   - รุ่นที่ใช้ได้
   - หมวดหมู่ (เครื่องยนต์, ช่วงล่าง, ไฟฟ้า)
   - ราคาทุน
   - ราคาขาย
   - จำนวนคงเหลือ
   - จุดสั่งซื้อ (Reorder Point)
   - ผู้จำหน่าย
   - รูปภาพ
   - Location ในคลัง

2. การจัดการสต็อก:
   - รับเข้าอะไหล่
   - เบิกอะไหล่
   - ตรวจนับสต็อก
   - ปรับยอด
   - คืนอะไหล์ (กรณีไม่ใช้)

3. การแจ้งเตือน:
   - อะไหล่ใกล้หมด
   - อะไหล์หมด
   - ถึงจุดสั่งซื้อ

#### 2.5.2 การสั่งซื้ออะไหล่
1. การสร้างใบสั่งซื้อ:
   - รายการอะไหล์
   - จำนวน
   - ผู้จำหน่าย
   - ราคา
   - วันที่สั่ง
   - กำหนดส่ง
   - สถานะ (สั่งแล้ว, รับแล้ว, ยกเลิก)

2. การรับอะไหล์เข้า:
   - ตรวจรับสินค้า
   - ตรวจนับ
   - เปรียบเทียบกับใบสั่งซื้อ
   - บันทึกรับเข้า
   - อัพเดตสต็อก

3. การจัดการผู้จำหน่าย:
   - ข้อมูลผู้จำหน่าย
   - เบอร์โทร
   - ที่อยู่
   - เงื่อนไขการชำระเงิน
   - ประวัติการสั่งซื้อ

### 2.6 การคำนวณค่าบริการ
#### 2.6.1 ค่าใช้จ่าย
1. ค่าแรง:
   - ค่าแรงช่างตามชั่วโมง
   - ค่าแรงตามงาน (Job Rate)
   - ค่าแรงวินิจฉัย
   - ค่าบริการเหมาจ่าย

2. ค่าอะไหล์:
   - ราคาอะไหล์ที่ใช้
   - จำนวน
   - รวมเป็นเงิน

3. ค่าบริการอื่นๆ:
   - ค่าบริการล้างรถ
   - ค่าจอดรถ (กรณีซ่อมนาน)
   - ค่าเก็บรถ

4. ส่วนลดและภาษี:
   - ส่วนลดลูกค้า VIP
   - ส่วนลดโปรโมชัน
   - ส่วนลดประกันภัย
   - ภาษีมูลค่าเพิ่ม (VAT)

#### 2.6.2 การคำนวณ
1. การคำนวณอัตโนมัติ:
   - คำนวณค่าแรง
   - คำนวณค่าอะไหล์
   - คำนวณส่วนลด
   - คำนวณภาษี
   - รวมเป็นยอดสุทธิ

2. การแสดงรายละเอียด:
   - รายการค่าใช้จ่ายทั้งหมด
   - รายการอะไหล์
   - รายการค่าแรง
   - ส่วนลด
   - ยอดรวม

### 2.7 การส่งมอบรถ
#### 2.7.1 ขั้นตอนการส่งมอบ
1. ก่อนส่งมอบ:
   - ตรวจสอบงานเสร็จสมบูรณ์
   - ทดสอบการทำงาน
   - ล้างรถ
   - ดูดฝุ่นภายใน
   - แจ้งลูกค้าพร้อมรับรถ

2. การส่งมอบ:
   - แจ้งรายการงานที่ทำ
   - แจ้งอะไหล์ที่เปลี่ยน
   - แจ้งข้อควรระวัง
   - แจ้งข้อแนะนำ
   - มอบอะไหล์เก่า (ถ้าลูกค้าขอ)
   - ให้ลูกค้าตรวจสอบ
   - ลงนามรับรถ

3. เอกสารการส่งมอบ:
   - ใบรับรถ
   - ใบเสร็จรับเงิน/ใบกำกับภาษี
   - รายละเอียดงานที่ทำ
   - ใบรับประกันงาน
   - คู่มือการดูแล

#### 2.7.2 การรับประกันงาน
1. เงื่อนไขการรับประกัน:
   - ระยะเวลารับประกัน
   - ขอบเขตการรับประกัน
   - เงื่อนไขที่ไม่รับประกัน

2. การจัดการการรับประกัน:
   - บันทึกการรับประกัน
   - ติดตามอายุการรับประกัน
   - แจ้งเตือนหมดประกัน

### 2.8 การชำระเงิน
#### 2.8.1 วิธีการชำระเงิน
1. ช่องทางชำระเงิน:
   - เงินสด

   - ใบเสร็จรับเงินเต็มจำนวน

#### 2.8.2 การจัดการลูกหนี้

## 3. ความต้องการที่ไม่ใช่ฟังก์ชันการทำงาน
### 3.1 ประสิทธิภาพ
1. รองรับผู้ใช้งานพร้อมกันได้อย่างน้อย 20 คน
2. เวลาตอบสนองการค้นหาไม่เกิน 2 วินาที
3. เวลาคำนวณค่าบริการไม่เกิน 1 วินาที
4. ทำงานได้ 24 ชั่วโมง
5. รองรับการอัพโหลดรูปหลายรูปพร้อมกัน
6. สามารถทำงานแบบ Offline ได้บางส่วน

### 3.2 ความปลอดภัย
1. ระบบยืนยันตัวตนผู้ใช้งาน
2. เข้ารหัสข้อมูลส่วนบุคคล
3. เข้ารหัสข้อมูลการเงิน
4. บันทึกประวัติการใช้งานระบบ
5. สำรองข้อมูลอัตโนมัติทุกวัน
6. กำหนดสิทธิ์การเข้าถึงข้อมูล
7. Session Timeout หลังไม่ใช้งาน 30 นาที

### 3.3 ความสามารถในการใช้งาน
1. ใช้งานง่าย เป็นภาษาไทย
2. รองรับการทำงานบนเว็บเบราว์เซอร์
3. มี Mobile App สำหรับลูกค้าและช่าง
4. มีคู่มือการใช้งาน
5. มีระบบช่วยเหลือออนไลน์
6. แสดงผลได้ทุกขนาดหน้าจอ
7. รองรับ QR Code และ Barcode Scanner

### 3.4 ความน่าเชื่อถือ
1. System Uptime 99%
2. มีระบบสำรองข้อมูล
3. สามารถกู้คืนระบบได้ภายใน 2 ชั่วโมง
4. มีระบบแจ้งเตือนเมื่อเกิดข้อผิดพลาด
5. ทำงานได้แม้อินเทอร์เน็ตขัดข้อง (บางฟังก์ชัน)

## 4. ข้อจำกัดของระบบ
1. ต้องเชื่อมต่ออินเทอร์เน็ตสำหรับการนัดหมายออนไลน์
2. รองรับเว็บเบราว์เซอร์รุ่นใหม่เท่านั้น
3. รูปภาพต้องมีขนาดไม่เกิน 10MB ต่อรูป
4. เก็บข้อมูลย้อนหลัง 7 ปี
5. ต้องปฏิบัติตาม พ.ร.บ. คุ้มครองข้อมูลส่วนบุคคล (PDPA)
6. แอปพลิเคชันรองรับ iOS 12+ และ Android 8.0+
7. ต้องมีเครื่องพิมพ์สำหรับพิมพ์ใบรับรถและใบเสร็จ
8. ต้องมี Barcode/QR Code Scanner สำหรับสแกนอะไหล์
